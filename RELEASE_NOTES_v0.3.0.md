# AI Design Rules v0.3.0

## Expressive Systems And Agent Context

This release expands AI Design Rules from foundational product structure into evidence-backed visual direction, systematic motion, and observable agentic workflows.

## Included

- Seven dated observations drawn from current Apple, Google, Chrome, and Figma guidance.
- Three research objects:
  - Expressive System UI In 2026;
  - Motion As State Continuity;
  - Contextual And Observable Agentic Interfaces.
- Three seed rules:
  - `VIS-002` Keep Expression Subordinate To Content;
  - `UX-004` Use Motion To Explain State Continuity;
  - `UX-005` Expose Agent Work And Intervention.
- A `reference-driven-design` skill for comparative, traceable visual reference briefs without copying brands.
- Expanded visual, motion, interaction, and prompt-architecture skills.
- Prototype Review `0.2.0` with explicit state, responsive, accessibility, motion, and agent lifecycle coverage.
- Design QA `0.2.0` with the new rule coverage and platform fallback checks.
- Research-led context resolution that includes downstream rules, prompts, and review gates.

## Validation

- Knowledge metadata and registry relationships are validated.
- Generated indexes and graph diagnostics are synchronized.
- DesignLint v0 reports no missing evidence-chain relationships.
- Automated context and validation regression tests pass.

## Evidence Boundary

The new research and rules are `draft` and `seed`.

They are supported by traceable primary-source observations, but have not yet been validated by paired rendered benchmark runs or independent evaluation. The release improves what agents can retrieve, apply, and review; it does not claim measured visual-quality improvement.

The existing Todo benchmark remains directional.

## Recommended Next Validation

1. Add rendered benchmark scenarios for:
   - consumer utility;
   - dense operations or fintech;
   - agentic workspace.
2. Capture baseline and graph-backed outputs at desktop and mobile viewports.
3. Store screenshots, raw prompts, evaluator identity, and review notes.
4. Promote rules beyond `seed` only after evidence-backed review.
