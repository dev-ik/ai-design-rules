# Prompt Index

Prompts apply patterns and rules to a specific agent task.

Prompts should not introduce new design advice. They should reference existing patterns and rules.

## Current Prompts

- `PROTOTYPE_REVIEW.md` - reviews prototypes before refactoring.

## Future Metadata

Prompt objects should follow the metadata model in `docs/KNOWLEDGE_ENGINE.md`.

Each prompt should define:

- stable ID;
- status;
- version;
- required patterns;
- required rules;
- last reviewed date.

Manual indexes are transitional. Future indexes should be generated from registry metadata.
