# Things 3

## Why Study It

Things 3 is useful for studying calm productivity UX, typography, spacing, progressive disclosure, and list editing that feels lightweight on mobile and desktop.

## Source Signals

- Things is a personal task manager for planning the day, managing projects, and making progress.
- The product uses Today, This Evening, Upcoming, headings, checklists, reminders, Quick Find, and direct manipulation.
- Things emphasizes hiding extra task fields until they are needed.

Sources:

- https://culturedcode.com/things/
- https://culturedcode.com/things/features/

## Observed Design Behavior

- The Today view turns a large task database into a daily plan.
- Extra fields such as tags, checklist, start date, and deadline are available without dominating the to-do.
- This Evening separates later tasks from the main day without creating a new system.
- Upcoming combines scheduled tasks, repeating tasks, deadlines, and calendar events.
- Direct manipulation is central: drag to insert, reorder, move, and structure.

## Design Takeaways For Agents

- Separate capture, planning, and doing.
- Hide power until the user needs it.
- Use time language that matches user intent: today, evening, upcoming, someday.
- Make list editing feel tactile, not like form administration.

## Candidate Rule Directions

- Daily planning screens should reduce the total system into today's useful subset.
- Advanced metadata should be progressively disclosed.
- Mobile list editing should support fast insertion, selection, and reordering.

## Do Not Copy Blindly

- Do not copy Things' minimalism into collaboration-heavy products that need visible ownership and status.
- Do not hide metadata if users must compare or audit items quickly.
