# Chat Benchmark

## Product Brief

Create a modern consumer chat surface for a small trusted group.

The app should help users read recent messages, send a message quickly, understand unread state, and inspect a conversation without losing context.

## Expected User

- Consumer coordinating with a small trusted group.
- Uses mobile frequently.

## Expected Platform

- Responsive web app.
- Mobile-first behavior required.

## Constraints

- No enterprise admin tools.
- No public channels.
- No bot marketplace.
- Include empty, loading, failed-send, and offline states.
- Keep message sending fast and clear.

## Evaluation Criteria

- Product Thinking
- UX
- Information Architecture
- Navigation
- Accessibility
- Mobile-first
- State Design
- Performance Awareness
- Overall Product Quality
