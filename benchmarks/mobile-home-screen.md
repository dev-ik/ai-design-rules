# Mobile Home Screen Benchmark

## Product Brief

Create a mobile home screen for a consumer productivity app.

The screen should help users understand what needs attention today, take the primary action quickly, and navigate to important areas without feeling like a dashboard.

## Expected User

- Consumer opening the app repeatedly during the day.
- Wants quick orientation and one obvious next action.

## Expected Platform

- Mobile web app at approximately 390px width.
- Desktop is not required for this benchmark.

## Constraints

- No analytics cards.
- No dense admin dashboard.
- Include empty, loading, and error states.
- Primary action must be reachable.
- Navigation must not bury the main daily action.

## Evaluation Criteria

- Product Thinking
- UX
- Information Architecture
- Navigation
- Accessibility
- Mobile-first
- Visual Hierarchy
- State Design
- Simplicity
- Overall Product Quality
