# Settings Benchmark

## Product Brief

Create a modern consumer settings experience for account, notifications, privacy, and preferences.

The app should help a user find a setting, understand consequences, change it safely, and recover from failed save states.

## Expected User

- Consumer configuring a personal or household app.
- Needs clarity and trust more than density.

## Expected Platform

- Responsive web app.
- Must work on mobile and desktop.

## Constraints

- No admin dashboard.
- No advanced developer settings.
- Include confirmation or recovery for risky changes.
- Include loading, error, disabled, and saved states.
- Privacy-related settings must be understandable.

## Evaluation Criteria

- Product Thinking
- UX
- Information Architecture
- Navigation
- Accessibility
- Mobile-first
- State Design
- Consistency
- Overall Product Quality
