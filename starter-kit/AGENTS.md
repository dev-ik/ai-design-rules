# Project AGENTS

This repository follows AI Design Rules.

## Agent Workflow

For every feature:

1. Read `docs/PRD.md`.
2. Read `docs/PERSONAS.md`.
3. Read `docs/USER_FLOWS.md`.
4. Read `docs/INFORMATION_ARCHITECTURE.md`.
5. Select applicable rules from AI Design Rules.
6. Select applicable patterns from AI Design Rules.
7. Explain the design reasoning before implementation.
8. Prototype or implement the smallest useful change.
9. Review with `reviews/DESIGN_REVIEW.md`.
10. Benchmark meaningful product changes.

## Do Not

- Do not jump directly from a feature request to code.
- Do not invent new design rules inside the product repo.
- Do not add UI complexity without a user goal.
- Do not skip empty, loading, error, disabled, and success states.
- Do not claim benchmark improvement without stored evidence.

## Output

When completing UI or product work, report:

- rules used;
- patterns used;
- states covered;
- mobile assumptions;
- accessibility checks;
- validation result.
