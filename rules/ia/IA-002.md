# IA-002 Define The Object Model Before Screens

Status: Draft
Domain: IA

## Source Research

- `research/products/linear.md`
- `research/products/telegram.md`
- `research/products/arc.md`

## Why

Agents often jump into screen layout before defining what the product objects are. This creates navigation, naming, and state inconsistencies.

## Rule

Before designing screens, define the core objects, their states, and how users move between them.

## Good

- Core objects have clear names and stable relationships.
- Screens expose the same object model from different views.
- Statuses and ownership labels mean the same thing everywhere.
- Navigation mirrors how users think about the work.

## Bad

- Similar concepts use different names across screens.
- A view creates duplicate data instead of another perspective on the same object.
- Status, ownership, and grouping change meaning by screen.

## Review

- List the product's core objects.
- List the states each object can enter.
- Check whether every screen maps to those objects and states.
- Check whether users can switch views without losing the underlying data.
