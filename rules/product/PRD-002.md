# PRD-002 Low-Friction Capture Before Organization

Status: Draft
Domain: Product

## Source Research

- `research/products/apple-reminders.md`
- `research/products/things-3.md`
- `research/products/telegram.md`

## Why

Everyday products often fail when they ask users to organize information before capturing it. Capture should be fast; organization can happen after the user has recorded the intent.

## Rule

Let users add the basic item first, then offer categorization, scheduling, metadata, or smart organization as follow-up actions.

## Good

- A user can create the core item with one clear action.
- Optional fields do not block capture.
- Smart suggestions are correctable and do not replace user control.
- Organization improves retrieval without changing the basic model.

## Bad

- The creation flow requires category, priority, project, tags, and dates before saving.
- Smart categorization silently changes user intent.
- Empty states explain the system instead of helping the user add the first item.

## Review

- Try creating the simplest possible item.
- Count how many required decisions happen before save.
- Check whether optional metadata can be added later.
- Check whether automation is reversible.
