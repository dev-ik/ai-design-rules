# VIS-001 Semantic Tokens Only

Status: Draft
Domain: Visual

## Source Research

- `research/products/linear.md`
- `research/products/things-3.md`

## Why

Visual consistency becomes fragile when agents choose one-off colors and spacing values. Semantic tokens preserve meaning across states, themes, and repeated surfaces.

## Rule

Use semantic tokens for color, spacing, typography, and state styling instead of raw one-off values.

## Good

- Color names describe purpose, such as text, surface, border, accent, success, warning, danger.
- Spacing follows existing scale values.
- Status colors carry the same meaning across screens.
- New styles reuse existing tokens before adding new ones.

## Bad

- Similar states use unrelated colors.
- Hex values are added directly to many components.
- A visual change requires editing unrelated screens one by one.
- Status colors change meaning by page.

## Review

- Search for raw colors and one-off spacing.
- Confirm each visual value maps to a semantic purpose.
- Check light and dark themes if they exist.
- Check whether status color meaning stays stable across the product.
