# Rules Index

Rules are reviewable design instructions derived from research.

Use this folder after `research/` and before `patterns/`.

```text
observations -> research -> knowledge -> rules -> patterns -> prompts -> reference projects -> reviews
```

## Rule Domains

- `product/` - product intent, repeated actions, task priority.
- `ia/` - information architecture, object models, navigation structure.
- `ux/` - interaction flow, progressive disclosure, mobile behavior.
- `visual/` - visual system decisions that support product clarity.
- `accessibility/` - accessibility requirements for usable interaction.

## Contribution Rule

Do not add a rule unless it can point back to research. If the source is only taste, keep it out of `rules/`.

Future rule metadata should follow `docs/KNOWLEDGE_ENGINE.md`.

Manual indexes are transitional. Future indexes should be generated from registry metadata.
