# UX-005 Expose Agent Work And Intervention

Status: Draft
Domain: UX

## Source Research

- `research/ux/contextual-agentic-interface.md`

## Why

For delegated or long-running work, a prompt thread alone does not reliably communicate progress, scope, required input, or recovery. Users need enough observable state to supervise outcomes without managing every internal step.

## Rule

When an agent performs delegated, long-running, or interruptible work, expose its user-relevant status, scope, result, and intervention points outside transient conversational output.

## Good

- Work distinguishes queued, active, blocked, review-ready, complete, failed, and cancelled states when applicable.
- The user can inspect sources, scope, output, and the next required decision.
- Risk-appropriate cancel, retry, edit, or review actions are available.
- The primary product goal stays more prominent than agent mechanics.

## Bad

- A loading message is the only record of a long-running task.
- Users cannot tell whether work is active, blocked, complete, or failed.
- The interface exposes internal reasoning instead of actionable state.
- Agent controls dominate an otherwise simple instant action.

## Review

- List the actual lifecycle states for the delegated work.
- Verify that status and next action remain visible after leaving the chat.
- Test failure, cancellation, retry, and human-input paths.
- Confirm the interface reveals useful evidence without exposing sensitive or irrelevant internals.
