# UX-002 Progressively Disclose Power

Status: Draft
Domain: UX

## Source Research

- `research/products/things-3.md`
- `research/products/apple-reminders.md`
- `research/products/telegram.md`

## Why

Power features help advanced users but can slow down first use. Progressive disclosure keeps the default path simple while preserving depth.

## Rule

Show the minimum controls needed for the current task, and reveal advanced fields, filters, or modes only when they become relevant.

## Good

- The default state supports basic completion without explanation.
- Advanced metadata is discoverable but not visually dominant.
- Users can add detail after capture.
- The interface remains usable without learning every feature.

## Bad

- Every possible field appears in the first form.
- Advanced filters are required before the first useful result.
- The UI hides essential status that users need for comparison or audit.

## Review

- Identify which fields are required for the first useful action.
- Move non-required metadata behind edit, details, filter, or secondary controls.
- Check that hidden controls are still discoverable.
- Check that hiding information does not harm collaboration or accountability.
