# A11Y-003 Visible Keyboard Focus

Status: Draft
Domain: Accessibility

## Source Research

- `research/accessibility/keyboard-focus-and-interaction-motion.md`

## Why

Keyboard users cannot operate a flow reliably when focus disappears or its indicator is visually hidden. Focus must communicate the current interaction target for as long as it remains active.

## Rule

Every keyboard-operable control must retain a visible focus indicator while it has keyboard focus. Do not remove or render the platform focus state invisible without providing an equally visible replacement.

## Good

- A focused input, button, task row, or sheet close control has an indicator that remains visible.
- A custom control uses a visible `:focus-visible` state or retains the platform default.
- Keyboard focus remains apparent through validation and recovery states.

## Bad

- A global outline reset leaves keyboard users with no focus indication.
- Focus appears briefly, then fades while the control remains active.
- A dialog opens without a visible focused control or an understandable keyboard path.

## Review

- Navigate the primary flow using only a keyboard.
- Verify each interactive control has a visible focus state and that it persists.
- Check focus during loading, validation error, detail open, and close actions.
