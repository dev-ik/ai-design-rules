# AI Design Rules v0.1.0

AI Design Rules is an AI-first knowledge base for helping coding agents build products, not dashboards.

This first release establishes the foundation: schema-backed knowledge objects, typed relationships, generated navigation, validation tooling, and an evidence workflow for measuring whether the rules improve AI-generated products.

## Included

- Schema-first metadata model for knowledge objects.
- Registry-backed research, rules, patterns, and prompts.
- Generated indexes and graph report.
- Validation script and `npm run check`.
- Benchmark methodology and evaluation rubric.
- First Todo App benchmark evidence.
- OSS cleanup with personal reference projects removed.

## Intentionally Not Included

- A UI kit or component library.
- A Tailwind preset or design token package.
- DesignLint implementation.
- A large rule or pattern catalog.
- Public reference projects.

## Current Limitations

- The knowledge graph is still small.
- Manual registry updates are still required.
- Generated indexes are transitional until more metadata is migrated.
- Skills are documented but not yet schema-migrated.
- Reference projects are omitted from the first OSS release.

## Benchmark Caveat

The first benchmark run is directional, not conclusive. It proves the evidence workflow can be used, not that AI Design Rules has statistically validated product-quality gains.

Future claims should be based on repeated benchmark runs across scenarios, agents, and reviewers.

## Next Roadmap

- Expand benchmark coverage and independent evidence.
- Migrate skills and checklists into the schema-first graph.
- Improve registry automation.
- Add generated contribution reports.
- Prepare DesignLint requirements without implementing it prematurely.
