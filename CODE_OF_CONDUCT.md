# Code of Conduct

AI Design Rules is a public OSS project for practical, evidence-driven product design knowledge.

## Expected Behavior

- Be direct, respectful, and specific.
- Critique ideas, files, and claims instead of people.
- Explain reasoning when proposing changes.
- Keep discussions focused on repository quality and public usefulness.
- Welcome good-faith contributions from people with different levels of design, engineering, and AI tooling experience.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeated low-effort comments that derail maintainers or contributors.
- Presenting unsupported claims as evidence.

## Evidence Standard

This project values benchmark-backed and source-backed claims. If a discussion concerns product quality, design rules, or AI output improvement, prefer reproducible evidence over preference.

## Enforcement

Maintainers may edit, close, hide, or remove issues, pull requests, comments, or accounts that violate this code of conduct.

For security-sensitive reports, use `SECURITY.md` instead of public issues.
