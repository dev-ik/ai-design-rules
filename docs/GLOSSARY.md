# Glossary

## Rule

A reviewable design instruction. A rule explains why it exists, what good output looks like, what bad output looks like, and how to review the result.

## Pattern

A reusable product or UI solution derived from one or more rules. A pattern should describe when to use it, when not to use it, and what tradeoffs it carries.

## Skill

A packaged agent workflow that applies stable rules, prompts, and review steps to a recurring task.

## Prompt

Task-specific instruction text that helps an AI agent apply the knowledge base to a product, screen, or review. A prompt should guide decisions, not replace reasoning.

## Review

A structured check of an agent's output against product goals, rules, patterns, accessibility expectations, and implementation constraints.

## Design QA

The final quality check for design output. Design QA looks for product fit, hierarchy, spacing, responsive behavior, accessibility, and rule coverage.

## Reference Project

A real or realistic product archetype used to validate whether the rules work in practice. Public reference projects are omitted from the first OSS release and should be backed by benchmark evidence when added.

## Product Thinking

Reasoning from the user's goal, context, repeated action, and product value before choosing UI structure or visual treatment.

## UX

User experience: the flow, feedback, states, and interaction model that determine whether a person can complete a task with low effort.

## IA

Information architecture: how information, navigation, hierarchy, and destinations are organized so the product is understandable.

## Design System

The reusable foundation for visual and interaction decisions. In this repository, design system guidance supports rules and product outcomes; it is not a component library.
