# Mobile First

Mobile-first means the product must work in the smallest common consumer context before desktop space is used for support.

Use `390px` width as the first-pass constraint for consumer workflows.

## Agent Checklist

- Is the primary action visible without horizontal scanning?
- Can the main action be reached in the thumb zone?
- Are tap targets at least `44x44` CSS pixels?
- Does navigation stay understandable without desktop sidebars?
- Do long labels wrap or compress without breaking layout?
- Are empty, loading, and error states usable on mobile?

## Desktop Rule

Desktop should add context, comparison, or speed. It should not change the core product model that mobile users rely on.
