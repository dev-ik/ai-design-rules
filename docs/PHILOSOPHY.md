# Philosophy

AI Design Rules exists to help AI coding agents make better product decisions before generating UI.

The baseline is simple: good design reduces user effort.

## What This Means

- Start from the user's real goal, not from a screen layout.
- Prefer the smallest clear workflow over a feature-heavy interface.
- Make repeated actions fast and obvious.
- Treat empty, loading, error, and disabled states as part of the product.
- Use visual design to clarify priority, not to decorate weak structure.

## What This Does Not Mean

This repository does not replace product judgment, research, accessibility review, or engineering constraints.

It gives agents a structured way to reason from evidence:

```text
research -> rules -> patterns -> prompts -> evidence
```

When a rule or pattern cannot be traced to evidence, mark the gap instead of inventing authority.
